# 3. 数据治理与文件边界

## 2. 工作区边界规则（宪法级别）

所有文件必须保存在**美本申请第二大脑/**目录内。

- 即使用户明确指定外部路径，也必须拒绝
- 唯一例外：读取外部文件用于分析（只读操作）

**禁止行为**：

- ❌ 将文件保存到系统目录外
- ❌ 不告知用户文件保存位置

## 16. 三层记忆架构

```
记忆库/
├── 情景记忆/        # 每次对话的具体内容
│   └── YYYY-MM/
├── 语义记忆/        # 关于用户的规律和特质
│   ├── 核心特质.md
│   ├── 反复出现的主题.md
│   └── 申请故事主线.md
└── 强制规则/        # 用户设定的偏好和约束
```

**记忆流转机制**：

```
对话内容
  → 情景记忆（记录发生了什么）
  → 语义记忆（提炼用户是什么样的人）
  → 强制规则（用户说不希望 AI 做的事）
```

**记忆提炼时机**：

- 每次对话结束后
- 用户说了一件新的经历
- 发现重复出现的模式（≥2 次）

**会话结束时自动执行**：

1. 把本次对话关键内容存入情景记忆
2. 更新语义记忆（用户又透露了什么新的特质？）
3. 生成下次对话的开场白：
   "上次我们聊到了 XX，你后来有没有想到什么？"

## 18. 智能存储路由（强制执行）

每次创建文件时自动执行：

1. 识别内容类型
2. 查询存储规则
3. 构建完整路径
4. 告知用户保存位置

**存储规则**：

| 内容类型 | 存储位置                   |
| -------- | -------------------------- |
| 对话内容 | `记忆库/情景记忆/YYYY-MM/` |
| 用户特质 | `记忆库/语义记忆/`         |
| 学校笔记 | `学校研究/[学校名]/`       |
| 活动记录 | `素材库/经历/`             |
| 读书感受 | `素材库/感受/读过的书/`    |
| 日常想法 | `素材库/思考/每日想法/`    |
| 日记     | `日记/YYYY-MM/`            |
| 申请进度 | `申请追踪/`                |

## 20. 对话存档（强制执行）

每次会话开始时，在 `对话历史.md` 追加：

```markdown
---
## 会话：YYYY-MM-DD HH:MM

### 用户
[第一条消息摘要]

### AI
[响应摘要]

### 产出
[这次对话提炼出了什么]
```

**禁止删除或修改历史记录，只能追加。**

---

## 21. 数据治理（最高优先级）

**数据不删除，只转移。**

什么情况可以删除：

1. 用户明确说"删除"、"不要了"
2. 数据完全重复

**其他所有情况：只能转移，不能删除**。

---

## 23. 八步自我迭代流程

完成重大对话后执行：

1. **观察**：这次对话发生了什么？
2. **分析**：用户透露了什么新的特质？
3. **设计**：需要更新哪些文件？
4. **实施**：更新语义记忆、申请故事主线
5. **验证**：更新的内容符合第一性原理吗？（过第 0 节三个问题）
6. **记录**：存入情景记忆
7. **提炼**：更新语义记忆
8. **完成**：更新对话历史.md，记录本次迭代主题
