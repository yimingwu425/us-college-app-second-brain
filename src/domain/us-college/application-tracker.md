# 6. 申请追踪与学校研究约定

## 时间意图捕获

用户表达时间、计划、截止日期或申请批次时，需要识别为申请追踪线索。

触发词包括：

- "明天"、"下周"、"X 月 X 日"
- "要交"、"截止"、"deadline"、"ED"、"EA"、"RD"
- "计划"、"打算"、"记得"

相关事项进入 `申请追踪/Deadline总览.md`，并告知用户："已添加到申请追踪，届时会提醒你"。

## Deadline 总览

`申请追踪/Deadline总览.md` 是所有时间敏感事项的集中入口，按紧急程度维护：

```markdown
# Deadline 总览

## 🔴 紧急（一个月内）

## 🟡 本季度

## 🟢 长期规划
```

启动时读取 `申请追踪/Deadline总览.md` 检查近期待办。和 ED、EA、RD、材料提交、面试、奖学金、作品集等相关的事项，都应进入 Deadline总览。

## 学校研究

学校笔记保存在 `学校研究/[学校名]/`。用户分享学校官网、申请案例、文章链接，或主动记录某所学校信息时，提取核心信息、和用户人设对比，并判断是否需要加入申请追踪或学校研究。

每所学校的研究目录采用：

```text
学校研究/
└── 学校名/
    ├── 基本信息.md
    └── 我对这所学校的想法.md
```

学校研究的重点不是堆资料，而是帮助用户建立真实认知：他对这所学校具体了解多少？喜欢这所学校的什么？

## 每所学校进度

申请进度保存在 `申请追踪/`，其中：

```text
申请追踪/
├── Deadline总览.md
└── 每所学校进度/
```

`每所学校进度/` 用于承接单校维度的申请状态、材料进展、补充文书、面试、提交结果等信息。涉及某所学校的具体申请进展时，优先归入这里；涉及跨学校的截止日期时，同步进入 Deadline总览。
