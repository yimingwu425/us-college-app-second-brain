# 6. 申请追踪与学校研究约定

## 时间意图捕获

时间意图指和时间、计划、截止日期或申请批次相关的信息，常见表达包括：

- 相对或绝对日期："明天"、"下周"、"X 月 X 日"
- 截止语义："要交"、"截止"、"deadline"
- 申请批次：ED、EA、RD
- 计划语义："计划"、"打算"、"记得"

这些信息属于申请追踪域，用于组织 Deadline 总览和单校进度。

## Deadline 总览

`申请追踪/Deadline总览.md` 是所有时间敏感事项的集中入口，按紧急程度维护：

```markdown
# Deadline 总览

## 🔴 紧急（一个月内）

## 🟡 本季度

## 🟢 长期规划
```

和 ED、EA、RD、材料提交、面试、奖学金、作品集等相关的跨学校或跨事项截止日期，属于 Deadline总览。

## 学校研究

学校笔记保存在 `学校研究/[学校名]/`。学校官网、申请案例、文章链接、学校特点和个人匹配判断，都属于学校研究材料。

每所学校的研究目录采用：

```text
学校研究/
└── 学校名/
    ├── 基本信息.md
    └── 我对这所学校的想法.md
```

学校研究的重点不是堆资料，而是帮助用户建立真实认知：他对这所学校具体了解多少？喜欢这所学校的什么？

## 每所学校进度

申请进度保存在 `申请追踪/`，其中：

```text
申请追踪/
├── Deadline总览.md
└── 每所学校进度/
```

`每所学校进度/` 用于承接单校维度的申请状态、材料进展、补充文书、面试、提交结果等信息。它和 Deadline总览的区别是：前者按学校组织进度，后者按时间组织截止事项。
